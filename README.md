### 📌 Development Workflow

All team members should follow this workflow while contributing to the project.

### 1️⃣ Clone the Repository

```
git clone <repository-url>
cd gigshield
```

### 2️⃣ Create a Feature Branch

Always create a separate branch for your work.

```
git checkout -b feature/feature-name
```

Example:

```
git checkout -b feature/fraud-detection-module
```

### 3️⃣ Implement Changes

While working on the project:

- Write clean and readable code
- Add comments where necessary
- Follow the agreed project architecture
- Do not modify unrelated files

### 4️⃣ Commit Your Changes

```
git add .
git commit -m "feat: short description of the update"
```

Example:

```
git commit -m "feat: added weather trigger monitoring"
```

### 5️⃣ Push Your Branch

```
git push origin feature/feature-name
```

### 6️⃣ Create a Pull Request

Open a Pull Request to the **main branch** and include:

- Description of the feature
- Changes made
- Any screenshots or demo (if applicable)

At least **one team member review** is recommended before merging.

---

### ⚠️ Team Rules

- Do **not push directly to the `main` branch`**
- Always work through **feature branches**
- Keep commits small and meaningful
- Ensure new changes do not break existing functionality
- Communicate major changes with the team before implementing

These guidelines help keep the repository organized and prevent merge conflicts.
